# SQL Rows Close Example Decision

Commit: `6eb1ab0 Add SQL rows close example`

## Options

1. Only document SQL rows leaks in `BUGS.md`.
   - Score: 3/5
   - Low effort, but the repository stays less useful because readers cannot run the linter report.

2. Add one broad `enable: all` golangci-lint example.
   - Score: 2/5
   - It may find the issue, but it teaches people to accept noisy configs without understanding which linter matters.

3. Add a focused `sqlclosecheck` example.
   - Score: 5/5
   - It maps directly to the "unclosed rows" bug and keeps the diagnostic easy to read.

4. Add a combined `sqlclosecheck` plus `rowserrcheck` example.
   - Score: 3/5
   - Both bugs are useful, but combining them can hide one report behind same-line de-duplication and makes the example less crisp.

## Selected

Option 3.

The example checks `rows.Err()` and intentionally omits only `rows.Close()`, so the report is specifically about database resource lifetime. `rowserrcheck` and the `bufio.Scanner.Err()` gap are recorded in `BUGS.md` as separate follow-up material.
